On This Page

Introduction Core Concepts Installation Configuration Examples FAQ

Introduction

Tailwind CSS is a utility-first CSS framework that gives you low-level building blocks to construct any design directly in your markup. Unlike traditional frameworks, Tailwind does not ship with pre-built components — instead it gives you the tools to build your own.

This approach keeps your stylesheet lean, your designs consistent, and your workflow fast. Once you get comfortable with the class names, you rarely need to leave your HTML file.

Core Concepts

The core idea behind Tailwind is utility classes — single-purpose class names like flex, pt-4, and text-center that each do exactly one thing.

By combining these small classes you can build complex layouts without ever writing a line of custom CSS. The result is a highly consistent design system that scales effortlessly across an entire project.

Installation

The fastest way to get started is via the CDN script tag — no build step required. For production projects, install Tailwind as a PostCSS plugin via npm so the compiler can tree-shake unused styles and keep your bundle small.

npm install -D tailwindcss postcss autoprefixer

Configuration

Tailwind is configured via a tailwind.config.js file at the root of your project. Here you can extend the default theme, add custom colors, define breakpoints, and enable plugins.

The content array tells Tailwind which files to scan for class names so unused styles are safely purged in production builds.

Examples

The best way to learn Tailwind is by building real components. Start with a simple card, then progress to navbars, grids, modals, and full page layouts. Each exercise in this document targets a specific set of utilities so you build muscle memory for the most common patterns.

FAQ

Is Tailwind only for small projects? Not at all — Tailwind scales from personal sites to large enterprise applications. Companies like GitHub, Shopify, and Vercel use it in production.

Does it work with React or Vue? Yes, Tailwind is framework-agnostic. It works with any technology that outputs HTML, including React, Vue, Svelte, Angular, and plain HTML files.

15. Responsive Order Change

Why Order Matters in Layouts

In responsive design, the visual order of elements often needs to differ from the source order in HTML. Tailwind's order-* utilities let you control this without duplicating any markup.

On mobile, the image appears first so users immediately see what the section is about. On desktop, the text moves to the left and the image sits naturally on the right — all with a single class change.
